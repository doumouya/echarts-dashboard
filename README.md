# echarts-dashboard

![ci](https://github.com/doumouya/echarts-dashboard/actions/workflows/ci.yml/badge.svg)

A fully **client-side** dashboard: open a CSV, group + aggregate it, and chart it with ECharts — all
in the browser, the file never leaving the page. The aggregation engine is Rust compiled to
WebAssembly; the charts are drawn from numbers computed on-device. No server, no upload.

## Status
- **`dashboard-core`** — the pure aggregation engine: type-inferring CSV load + group/aggregate
  (count · sum · avg · min · max) → chart series. 5 tests, compiles to `wasm32`. ✅
- **`dashboard-wasm`** — the `WasmData` binding (`fromCsv` / `headers` / `kinds` / `aggregate`).
  Verified through wasm. ✅
- **`index.html`** — a working **single self-contained file**: ECharts + the wasm engine, inlined and
  instantiated in-page. Open it (no server) → drop a CSV → add chart cards (group by / aggregate /
  chart type), all in the browser. Verified end-to-end. ✅ *(The UI is a compact reference; the
  polished dashboard is the next step, built via Claude Design from [`docs/fe-brief.md`](docs/fe-brief.md).)*

## Layout
```
echarts-dashboard/
  index.html            # the built single-file demo (open it; generated by tools/build.sh)
  crates/core/          # dashboard-core — the pure aggregation engine
  crates/wasm/          # dashboard-wasm — the wasm-bindgen binding
  web/                  # the reference UI source (style.css, body.html, app.js)
  tools/build.sh        # fetches ECharts, compiles wasm, inlines everything into index.html
  docs/spec.md          # architecture + the wasm JS API contract
  docs/fe-brief.md      # the front-end brief (for Claude Design)
```

## Build / try
```sh
cargo test -p dashboard-core                 # the engine's behaviour
sh tools/build.sh                            # fetch ECharts + (re)build index.html  (--dev to skip wasm-opt)
# then just open index.html in any browser — no server, fully offline
```
